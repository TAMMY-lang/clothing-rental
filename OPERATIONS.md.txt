# 多金喜服装租赁平台日常运营手册

这份手册写给商家日常使用。你不需要懂代码，只需要按照页面上的按钮操作。

## 一、商家后台整体布局

商家登录后，会看到商家后台。

大致结构如下：

```text
┌──────────────────────────────┐ │ 顶部：店铺名称、刷新、退出登录 │ ├───────────┬──────────────────┤ │ 左侧菜单 │ 右侧内容区 │ │ 商品管理 │ 当前模块的表单和列表 │ │ 订单管理 │ │ │ 资金管理 │ │ │ 会员管理 │ │ │ 店铺设置 │ │ └───────────┴──────────────────┘


Plain Text


常用入口：

| 你想做什么 | 去哪里 |
|---|---|
| 发布商品 | 商家后台 → 商品管理 |
| 处理订单 | 商家后台 → 订单管理 |
| 查看收入 | 商家后台 → 资金管理 |
| 申请提现 | 商家后台 → 资金管理 |
| 修改公告 | 商家后台 → 店铺设置 |
| 配置试穿费用 | 商家后台 → 试穿设置 |
| 绑定 TOTP | 首页 → 账号安全 |

## 二、如何登录商家后台

### 1. 打开网站

打开你部署好的前端网址，例如：

```text
https://你 的店铺网址.vercel.app


Plain Text


### 2. 输入商家账号

默认商家账号：

```text
手机号：13900000000 密码：123456 短信验证码：123456


Plain Text


首次上线后，请立刻修改密码。

建议密码格式：

```text
Aa123456!


Plain Text


### 3. 绑定 TOTP

登录后找到：

```text
账号安全 → 绑定/重置 TOTP


Plain Text


操作方法：

1. 点击“绑定/重置 TOTP”。
2. 系统会显示一串密钥。
3. 打开 Google Authenticator 或 Authy。
4. 添加账号。
5. 扫描二维码或手动输入密钥。
6. App 会生成 6 位数字。
7. 把 6 位数字填回网页。

示意图：

```text
网页显示密钥/二维码 ↓ 手机身份验证器 App 扫码 ↓ App 显示 6 位动态码 ↓ 网页输入动态码 ↓ 绑定成功


Plain Text


## 三、如何发布商品

进入：

```text
商家后台 → 商品管理 → 发布商品


Plain Text


### 1. 填写商品基础信息

建议按下面顺序填写：

| 字段 | 怎么填 |
|---|---|
| 商品名称 | 例如：红色敬酒服、秀禾服、黑色晚礼服 |
| 分类 | 婚纱、礼服、旗袍、配饰等 |
| 风格 | 中式、轻奢、复古、甜美等 |
| 使用场景 | 婚礼、订婚、年会、拍摄等 |
| 吊牌价 | 商品原价 |
| 日租金 | 每天租赁价格 |
| 押金 | 用户需要支付的保证金 |

### 2. 上传商品图片

图片分两种：

```text
主图：商品列表和详情页最醒目的图片 详情图：用户点进商品后看到的更多图片


Plain Text


建议：

- 主图用干净背景。
- 图片清晰，不要太暗。
- 每张图片小于 5MB。
- 支持 jpg、png、webp。

### 3. 设置 SKU 规格

SKU 就是“颜色 + 尺码 + 库存”。

示例：

```text
红色 / S / 1 件 红色 / M / 2 件 香槟色 / L / 1 件


Plain Text


如果一个尺码只有一件，库存填 `1`。

### 4. 发布前检查

发布前检查：

- 商品名是否清楚。
- 图片是否显示。
- 日租金是否正确。
- 押金是否正确。
- 每个尺码库存是否正确。

## 四、如何处理订单

进入：

```text
商家后台 → 订单管理


Plain Text


订单状态大致流程：

```text
用户下单 ↓ 用户支付 ↓ 商家发货 ↓ 用户使用 ↓ 用户归还 ↓ 商家验收 ↓ 进入结算 ↓ 到账


Plain Text


### 1. 商家发货

当订单支付成功后，你需要发货。

操作：

1. 找到订单。
2. 点击发货或标记租赁中。
3. 填写快递公司。
4. 填写快递单号。
5. 保存。

注意：

- 快递单号要准确。
- 发货前确认商品尺码和颜色。
- 商品寄出前建议拍照留档。

### 2. 审核延期

如果用户申请延期：

1. 打开订单详情。
2. 查看延期天数和费用。
3. 判断商品后续档期是否冲突。
4. 如果不影响后续订单，可以同意。
5. 如果影响后续订单，建议拒绝并联系用户。

判断方式：

```text
是否已有后续预约 是否会影响其他用户使用 是否已经支付延期费用


Plain Text


### 3. 验收归还

用户归还后，你需要验收：

1. 检查衣服是否归还。
2. 检查是否有破损。
3. 检查是否需要清洁费。
4. 输入清洁费。
5. 点击验收通过。

资金规则：

```text
商家入账 = 租金 + 快递费 + 试穿费 + 清洁费 押金不进入商家余额 押金扣除清洁费后退给用户


Plain Text


示例：

```text
押金 200 元 清洁费 12 元 用户退回押金 188 元 商家收入增加清洁费 12 元


Plain Text


### 4. 快递拦截

如果出现特殊情况需要拦截快递：

常见情况：

- 用户下错尺码。
- 商品临时发现损坏。
- 用户取消订单但快递已发出。

处理方法：

1. 立刻联系快递公司。
2. 提供快递单号。
3. 申请拦截或退回。
4. 在订单备注中记录情况。
5. 主动联系用户说明。

平台目前不自动拦截快递，需要人工处理。

## 五、如何查看资金和提现

进入：

```text
商家后台 → 资金管理


Plain Text


你会看到：

| 项目 | 含义 |
|---|---|
| 可用余额 | 可以申请提现的钱 |
| 提现冻结 | 已申请提现但还没打款的钱 |
| 累计收入 | 历史总收入 |
| 最低提现 | 默认 100 元 |
| 提现手续费 | 当前默认 0 |

### 1. 什么时候到账

订单验收后进入结算。

默认规则：

```text
验收完成后 T+7 天到账


Plain Text


例如：

```text
7 月 1 日验收 7 月 8 日到账


Plain Text


### 2. 如何申请提现

1. 进入资金管理。
2. 确认可用余额大于 100 元。
3. 绑定提现方式。
4. 输入提现金额。
5. 提交申请。

注意：

- 低于 100 元不能提现。
- 提现申请后，金额会进入冻结。
- 平台确认打款后，提现完成。

## 六、如何管理会员等级

进入：

```text
商家后台 → 会员管理


Plain Text


会员等级一般用于给老客户优惠。

建议设置：

| 等级 | 条件 | 权益 |
|---|---|---|
| 普通会员 | 默认 | 无折扣 |
| 银卡会员 | 消费满一定金额 | 95 折 |
| 金卡会员 | 消费更高金额 | 9 折 |

设置时注意：

- 折扣不要太低，避免亏损。
- 可以先少量设置，运营一段时间后再调整。
- 活动期可以配合优惠券使用。

## 十、日常操作建议

每天：

- 查看新订单。
- 查看待发货订单。
- 查看待验收订单。
- 回复用户咨询。

每周：

- 检查库存是否准确。
- 更新商品图片。
- 查看资金是否可提现。

- 检查会员等级和优惠券。
- 导出或记录资金流水。
- 修改一次后台密码或检查 TOTP 是否正常。

## 十一、常见问题

### 1. 登录不上怎么办

先检查：

- 手机号是否正确。
- 密码是否正确。
- 是否需要 TOTP 动态码。
- 是否在另一台设备登录。

如果提示“账号已在其他设备登录”，重新登录即可。

### 2. TOTP 动态码总是错误

检查：

- 手机时间是否准确。
- 是否用了正确的身份验证器账号。
- 动态码是否过期。

动态码每 30 秒变化一次，快结束时不要输入，等下一组。

### 3. 商品图片上传失败

检查：

- 图片是否超过 5MB。
- 格式是否是 jpg、png、webp。
- 网络是否稳定。

### 4. 用户说押金没退

先看订单是否已经验收。

验收后系统会记录押金退款流水：

```text
DEPOSIT_REFUND


Plain Text


如果接入真实支付，还需要支付渠道实际退款。

### 5. 为什么订单钱不是立刻到账

平台默认：

```text
验收后 T+7 天到账


Plain Text


这是为了给售后和争议处理留时间。

### 6. 提现为什么失败

常见原因：

- 可用余额不足。
- 低于最低提现金额 100 元。
- 提现方式没有绑定。
- 管理员还没审核。

### 7. 商家账号可以多人共用吗

不建议。

系统开启了单设备登录，同一时间只允许一个设备保持登录。新设备登录后，旧设备会被下线。
